## Вход в контур

Через GitHub Issues доступны две структурированные формы:

- **Научный конструкт** — декларация, аксиома, гипотеза, теорема или оператор;
- **Инженерный эксперимент** — проверяемое применение ТзАр с конфигурацией, метриками и результатом.

Положения проходят статусы `draft → candidate → verified → canonical`. Канонический статус присваивается только автором ТзАр.

## Метрики проводимости

- `α` — соответствие;
- `IЯ` — интеграция ядра;
- `Cm` — контейнер смыслов;
- `Q` — качество отклика;
- `T` — текучесть.

Шкала: `0 / 0.25 / 0.5 / 0.75 / 1.0`.
